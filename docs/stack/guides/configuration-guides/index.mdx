This section covers configuration options for The Linea stack components. Configuration is organized
into common configurations (applicable to all deployment models) and specialized configurations
(specific to deployment models).

## Configuration approach

Following the Besu documentation pattern, common configurations are documented for public networks
where possible, with specialized configurations documented separately for private networks and
specific deployment models.

## Configuration files

The Linea stack uses TOML configuration files for most components:

- **Maru**: `maru-config.toml`
- **Linea Besu**: `linea-besu.config.toml`
- **Coordinator**: `coordinator-config.toml`

Configuration files are typically mounted as read-only volumes in Docker deployments.

## Environment variables

Some configuration can be provided via environment variables:

- JVM options: `JAVA_OPTS`
- Profile selection: `BESU_PROFILE`
- Network-specific settings

## Best practices

- Use profiles for common configurations
- Keep configuration files in version control
- Document custom configurations
- Test configuration changes in non-production environments
- Use configuration validation before deployment
